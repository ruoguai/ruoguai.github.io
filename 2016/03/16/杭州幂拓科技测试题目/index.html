  • 题目如下：
    学习能力比原有的老知识储备更被看重，给一个很小的测试
    任务，看你多久能完成，完成后代码离线文件传给我看看，我们就能大
    体上知道你的动手能力如何。如果感兴趣的话，请看下面的测试任务：

用C++编写一个小程序，利用OTL (http://otl.sourceforge.net/home.htm) 库
读取MetGS.accdb数据库中的ModelInfo表的内容，将各行各列打印在屏幕上。

看到这个测试题我的第一反应是个大写的懵！
我听说过STL一个标准的模板库，但是OTL还是第一次听说。
不管怎么样感谢给我发邮件的姐姐给我提供了OTL相关的信息，看了OTL的introduction知道了OTL用来做什么的。
在介绍里他们写到：
OTL 4.0 supports all major and a few minor relational database types (Oracle, MS SQL Server, DB2, Informix, TimesTen, MAX/DB, etc.) natively via the corresponding database API. For the rest of database types (PostgreSQL, MySQL, Sybase, SQLite, MS ACCESS, Firebird, etc.), OTL has support via ODBC 3.x, or ODBC 2.5 (legacy applications).

这些数据库的名字像Oracle、SQLServer、DB2是听说过的，并且曾经学习过SQLserver的相关软件和SQL语句。Access作为微软office的一部分，也只是简单的了解过，既然给的是Access文件，那么我想要求就是使用OTL库完成连接数据库–>读取表格内容–>打印到屏幕文件中的相关信息。
